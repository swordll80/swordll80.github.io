一、JSON 基础语法
1JSON 定义// JSON = JavaScript Object Notation // 轻量级数据交换格式, 语言无关 // 基于两个结构: 键值对集合 + 有序值列表RFC 4627
2基本示例{ "name": "Alice", "age": 25, "isStudent": false, "hobbies": ["reading", "coding"], "address": { "city": "Beijing", "zip": "100000" } }RFC 4627
3JSON 数据类型// 1. string → "hello" // 2. number → 42, 3.14, -10, 1.5e10 // 3. boolean → true / false // 4. null → null // 5. object → { "key": "value" } // 6. array → [1, 2, 3]RFC 4627
4字符串规则// 必须用双引号 (不能用单引号) // 转义: \" \\ \/ \b \f \n \r \t \uXXXX { "message": "He said \"Hello\"\nNew line" }RFC 4627
5数字规则// 整数: 无前导零 (不能 0123) // 小数: 必须有数字在小数点前后 // 指数: e 或 E, 后可跟 + 或 - // 不支持: NaN, Infinity, undefined // 不支持: 八进制, 十六进制RFC 4627
6JSON 格式规则// 1. 最外层可以是 {} 或 [] (RFC 7159+) // 2. 键必须是字符串 (双引号) // 3. 不能有尾随逗号 // 4. 不支持注释 // 5. 空白(空格/制表/换行)被忽略RFC 7159
二、JSON Schema
7Schema 基础{ "$schema": "https://json-schema.org/draft/2020-12/schema", "$id": "https://example.com/person.schema.json", "title": "Person", "type": "object", "properties": { "name": { "type": "string" }, "age": { "type": "integer", "minimum": 0 } }, "required": ["name"] }JSON Schema
8类型约束// "type": "string" / "number" / "integer" // "type": "object" / "array" / "boolean" / "null" // 可多类型: "type": ["string", "null"] // 字符串: "minLength", "maxLength", "pattern" (正则) // 数值: "minimum", "maximum", "multipleOf" // 数组: "minItems", "maxItems", "uniqueItems" // 对象: "minProperties", "maxProperties"JSON Schema
9枚举与常量{ "color": { "enum": ["red", "green", "blue"] }, "version": { "const": "1.0" } }JSON Schema
10组合约束{ "allOf": [{...}, {...}], // 全部满足 "anyOf": [{...}, {...}], // 任一满足 "oneOf": [{...}, {...}], // 恰好一个满足 "not": {...} // 不满足 }JSON Schema
11引用 ($ref){ "definitions": { "address": { "type": "object", "properties": {...} } }, "properties": { "home": { "$ref": "#/definitions/address" }, "work": { "$ref": "#/definitions/address" } } }JSON Schema
三、JSONPath
12JSONPath 表达式// $ → 根元素 // $.store.book[0] → 第一本书 // $.store.book[*] → 所有书 // $..author → 所有 author (递归) // $.store.book[?(@.price < 10)] → 筛选 // $.store.book[0,1] → 前两本 // $.store.book[-1] → 最后一本 // $.store.book[:2] → 前两本(切片) // $.store.book[?(@.isbn)] → 有 isbn 的书JSONPath

九、常见Bug与陷阱
33尾随逗号// 错误: {"a": 1, "b": 2,} → 解析失败! // JSON 标准不允许尾随逗号 // JavaScript 对象允许, 但 JSON 不允许-
34单引号字符串// 错误: {'key': 'value'} → 解析失败! // JSON 必须使用双引号-
35数字前导零// 错误: {"zip": 01234} → 解析失败! // 正确: {"zip": "01234"} (用字符串)-
36特殊值处理// NaN, Infinity, undefined → 无法序列化 // 需替换为 null 或字符串 // 日期需转为 ISO 8601 字符串-
37循环引用// const a = {}; const b = {a}; a.b = b; // JSON.stringify(a) → TypeError // 解决: 用 toJSON 或 flatted 库-
38大数精度丢失// JSON.parse('{"id": 9007199254740993}') // id 变为 9007199254740992 (JS Number 精度问题) // 解决: 用字符串传大数, 或 json-bigint 库-
39编码问题// JSON 标准要求 UTF-8 编码 // 确保文件保存为 UTF-8 without BOM // Python: json.dumps(data, ensure_ascii=False)-
十、编程建议与最佳实践
40使用 JSON Schema 验证// 确保 API 输入输出符合预期结构 // 工具: ajv (JS), jsonschema (Python)-
41不要信任输入 JSON// 始终做 try-catch 处理 parse 异常 // 验证必要字段存在且类型正确-
42美化输出用 indent// 生产环境压缩, 开发环境带缩进 // JSON.stringify(obj, null, 2)-
43大 JSON 用流式处理// 不要一次性 parse 大文件 // 用流式解析器: JSONStream, ijson // 或用 JSON Lines 格式-
44属性命名规范// 推荐 camelCase: "firstName" // 或 snake_case: "first_name" // 团队统一一种风格 // API 字段不要随意改名(破坏兼容性)-
45JSON vs XML// JSON: 更轻量, 更易读, Web 标准 // XML: 支持 Schema/DTD, 命名空间, XSLT // JSON 适合数据交换, XML 适合文档标记-
