## Confluence 同步冲突保护
- 问题：`confluence.html` 和 `confluence_beta.html` 同时打开时会互相覆盖 confluence/ 目录下的文件，导致数据丢失
- 方案：
  1. `getFileText()` 改为返回 `{ text, lastModified }` 对象，同时获取文件内容和最后修改时间
  2. 新增全局变量 `loadedFileLastModified`，记录当前编辑器加载文件时的最后修改时间
  3. `saveCurrentPage()` 增加同步检查逻辑：
     - **手动保存**（`force: true`，即点击保存按钮或 Ctrl+S）：跳过检查，直接保存
     - **自动保存**（`silent: true`，即切换节点时触发）：先读取文件当前 lastModified，如果比 `loadedFileLastModified` 新，说明文件已被另一个实例修改，拒绝保存并提示冲突
  4. 保存成功后更新 `loadedFileLastModified = Date.now()`
- 修改文件：`confluence.html`、`confluence_beta.html`

## Confluence Beta 冲突解决弹窗
- 所有保存路径（手动保存、自动保存切换节点、离开前保存）统一使用冲突检测 + 弹窗对比解决
- **detectConflict()**: 检查磁盘文件 lastModified 是否比 loadedFileLastModified 新，返回 `{ conflict, diskText, diskTime }`
- **resolveConflict()**: 返回 Promise，弹出左右对比弹窗让用户选择
- 弹窗布局：
  - 左侧 = 当前编辑中的版本（带时间戳标签：较旧/较新）
  - 右侧 = 磁盘文件版本（带时间戳标签）
  - 底部按钮：◀ 使用我的版本 / 使用磁盘版本 ▶ / 💾 保存当前编辑结果
  - "显示 HTML 源码" 复选框切换 textarea 样式
  - Ctrl+S 在弹窗内保存自定义编辑，Esc 关闭
- saveCurrentPage() 移除 force/silent 分支差异，统一检测冲突
- 冲突解决后自动重新加载编辑器内容并清除 dirty 标记
- 关闭弹窗（X按钮/Esc）取消保存操作
- **Bug修复**：冲突弹窗默认模式（HTML源码关闭）下错误显示HTML源码，改为显示去除标签后的纯文本：
  - 新增 `stripHtml()` 函数，用 DOM 解析去除 HTML 标签
  - `showConflictDialog()` 将原始 HTML 存入 `data-html` 属性，textarea 初始显示纯文本
  - `updateConflictView()` 切换时：切换到源码模式从 `data-html` 加载，切换到纯文本模式先保存编辑到 `data-html` 再 strip
  - 修复了初始化时 `updateConflictView()` 覆盖 `data-html` 的 bug（改为手动设置 class）
  - `conflictSaveCustomBtn` 根据当前模式读取：源码模式用 textarea.value，纯文本模式用 data-html
- **Tree.json 标题冲突**：新增页面标题冲突检测，在保存时检查内存 tree.json 中的标题是否与磁盘 tree.json 中的标题冲突：
  - 新增 `loadedTreeLastModified` 全局变量，在加载 tree.json 时记录时间戳
  - 新增 `detectTreeConflict(nodeId)`：读取磁盘 tree.json，比较 lastModified，若冲突则查找对应节点标题
  - 新增 `findNodeById(nodes, id)`：在任意节点树中递归查找节点
  - 冲突弹窗顶部新增标题冲突警告行（`#conflictTitleAlert`），橙色背景，显示"我的标题 vs 磁盘标题"
  - 两个切换按钮："用我的标题"（默认选中）/"用磁盘标题"，选中高亮蓝色
  - `resolveConflict()` 结果改为 `{ html, title }` 对象，同时传递 HTML 和标题选择
  - `saveCurrentPage()` 重构：先检测 tree.json 冲突和 HTML 冲突，有任一冲突则弹窗解决后再统一写入
  - 无冲突时顺序：先写 tree.json（含标题），再写 HTML 文件
- 修改文件：`confluence_beta.html`

## 冲突弹窗编辑器改为富文本（WYSIWYG）模式
- 用户反馈：冲突编辑框显示的是部分 CSS 源码（如 `body { margin: 0; ... }`），而非完整的 HTML 页面效果
- 修改方案：将冲突弹窗左右两边的 `<textarea>` 替换为 `contenteditable` 的富文本编辑器 div，与主编辑器外观一致
- HTML 结构变更：
  - 每个 `.conflict-pane` 内新增 `.conflict-editor-wrap` 包裹层
  - 同时包含 `.conflict-editor`（contenteditable div，WYSIWYG 模式）和 `.conflict-source-editor`（textarea，源码模式）
- CSS 变更：
  - 新增 `.conflict-editor-wrap`、`.conflict-editor`、`.conflict-source-editor` 样式
  - `.conflict-editor` 外观与主编辑器 `.editor` 一致：白色背景、圆角、阴影、行高 1.75
  - `.conflict-pane.source-view` 控制 WYSIWYG/源码模式的显示切换
  - 删除旧的 `.editor-preview` 和 `.html-preview` 样式
- JS 变更：
  - `showConflictDialog()`: 用 `innerHTML` 填充 contenteditable div，同时初始化 source textarea
  - `updateConflictView()`: 切换时在 contenteditable 和 textarea 之间同步数据
  - `conflictSaveCustomBtn`: 根据当前模式从 contenteditable（`innerHTML`）或 textarea（`value`）读取
  - `stripHtml()` 函数已不再使用但保留
- 修改文件：`confluence_beta.html`
