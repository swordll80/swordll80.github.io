## Confluence 同步冲突保护
- 问题：`confluence.html` 和 `confluence_beta.html` 同时打开时会互相覆盖 confluence/ 目录下的文件，导致数据丢失
- 方案：
  1. `getFileText()` 改为返回 `{ text, lastModified }` 对象，同时获取文件内容和最后修改时间
  2. 新增全局变量 `loadedFileLastModified`，记录当前编辑器加载文件时的最后修改时间
  3. `saveCurrentPage()` 增加同步检查逻辑：
     - **手动保存**（`force: true`，即点击保存按钮或 Ctrl+S）：跳过检查，直接保存
     - **自动保存**（`silent: true`，即切换节点时触发）：先读取文件当前 lastModified，如果比 `loadedFileLastModified` 新，说明文件已被另一个实例修改，拒绝保存并提示冲突
  4. 保存成功后更新 `loadedFileLastModified = Date.now()`
- 修改文件：`confluence.html`、`confluence_beta.html`

## 唤醒时自动冲突检测
- 用户需求：页面从后台恢复时（如切换标签页回来），检查磁盘文件是否被外部修改，若有冲突则弹出冲突处理对话框
- 实现：在 `initEvents()` 内添加 `visibilitychange` 事件监听
  - 仅在 `!document.hidden`（页面变为可见）时触发
  - 前置条件检查：无正在显示的冲突对话框、有目录授权、`loadedFileLastModified > 0`、有当前编辑节点
  - 调用 `getFileText(fileName)` 重新读取磁盘文件，比较 `lastModified`
  - 若磁盘文件更新，同时检测 `tree.json` 标题冲突
  - 调用 `resolveConflict()` 弹出冲突对话框（与保存时用的同一个）
  - 用户选择后：写 tree.json、写 HTML、更新编辑器、更新时间戳
  - 用户取消时：提示"磁盘文件已被外部修改"
- 修改文件：`confluence_beta.html`

## 冲突对话框按钮改用原始快照
- 问题：用户点击"使用磁盘版本"后，页面刷新为编辑版本而非磁盘版本
- 原因：按钮从 contenteditable 编辑器的 `innerHTML` 读取内容，浏览器可能对 contenteditable 中的 HTML 做了规范化/剥离处理（如剥离 `<!doctype>`、`<html>`、`<head>` 等标签），导致读取到的内容不完整

## 字体应用后选中区域缩小 Bug 修复
- 问题：选中 3 行文字修改字体为楷体后，选中区域变成前 2 行
- 原因：`applyFontFamily()` 使用 `document.execCommand('fontName', ...)` 处理多段落选区时行为不一致——`<font>` 是行内元素，无法跨块级元素（如 `<p>`）正确包裹，浏览器会自动修复 DOM 结构导致选区塌缩
- 修复：放弃 `execCommand('fontName')`，新增 `wrapSelectionWithFontFamily(value)` 函数，手动遍历选区范围内的所有文本节点，为每个文本节点的选中部分包裹 `<span style="font-family:...">`，处理完后恢复选区范围
  - 使用 TreeWalker + `range.intersectsNode()` 收集所有文本节点及其偏移
  - 反向遍历处理（避免前面的替换使后面节点引用失效）
  - 追踪第一个和最后一个创建的 span，用它们重建选区范围
- `applyFontSize()` 同样修复：新增 `wrapSelectionWithFontSize(value)` 函数，逻辑与 fontFamily 一致
- 修改文件：`confluence_beta.html`

## 格式刷文字颜色修复
- 问题：文字颜色没有通过格式刷起作用
- 原因有两个：
  1. **捕获端**：`captureFormatPainter()` 只检查 `current.style.color`（元素内联 style），但 `execCommand('foreColor')` 产生的是 `<font color="...">` 标签而非 `style="color:..."`，所以从 `<font>` 标签中无法捕获到颜色。此外也未使用 `getComputedStyle()` 检测实际渲染颜色。

- 修复1：`captureFormatPainter()` 颜色捕获改为三层 fallback：
  - 先检查 `<font color="...">` 属性
  - 再检查 `current.style.color`
  - 最后用 `getComputedStyle(current).color` 检测渲染颜色，通过新增的 `rgbToHex()` 转为 hex 格式
- 修复2：新增通用 `wrapSelectionWithStyle(styleProp, value)` 函数，用 TreeWalker + `range.intersectsNode()` 为选区文本节点包裹 `<span style="prop:value">`，避免 execCommand 的跨块级元素问题
- 修复3：`applyFormatPainter()` 中 color 和 backgroundColor 改用 `wrapSelectionWithStyle()` 替代 `execCommand`
- 修改文件：`confluence_beta.html`

## 分析 confluence_beta 自动保存导致 c.html 数据丢失的根因
- 触发链路：切换到首页节点 → selectNode → saveCurrentPage → mergeIntoLoadedHtml
- 数据丢失的两个原因：
  1. **sanitizeHtml 白名单不全**：语法高亮 class 被移除（已修复）
  2. **DOMParser 序列化/反序列化循环**：`loadedHtml` → DOMParser → `.container.innerHTML` 替换 → `outerHTML` 输出，每次经过都会丢失原始格式（缩进、换行），这是 ~3KB 体积减小的主要原因
- c.html 等独立页面不适合用 confluence_beta 编辑器编辑（它们有自包含的 topbar/搜索/脚本）
