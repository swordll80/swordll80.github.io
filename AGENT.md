# swordll80.github.io 网站设计规范

## 网站定位

这是一个长期维护的个人网站，定位为：

- 个人简介与项目入口
- 在线工具集合
- 技术学习笔记
- 户外地图与轨迹分享

网站应保持简单、淡雅、一致，优先服务内容展示与工具入口，不追求复杂视觉效果。

## 总体原则

- 简单可靠，避免过度设计。
- 页面加载快速，减少第三方依赖。
- 桌面端与手机端都要可用。
- 导航结构清晰，目录层级不超过 3 级。
- 新增页面应继承统一布局、统一样式和统一交互。
- 优先使用原生 HTML、CSS、JavaScript。
- 图标优先使用 SVG 或纯 CSS 绘制，不依赖 PNG 图片。

## 主体布局

主页采用左右布局：

- 顶部：Header
- 左侧：Sidebar 目录
- 右侧：Main 内容区
- 底部：Footer

桌面端默认显示左侧目录。手机端默认隐藏目录，以获得更大阅读视野。

## 顶级目录

顶级节点固定为：

- 简介
- 工具
- 学习
- 户外

目录层级小于等于 3 级。

不再设置“古文”“聊斋志异”等目录节点。

## 目录交互

左上角提供目录按钮，用于显示或隐藏目录。

目录隐藏时：

- 桌面端：鼠标移动到左边缘热区时显示目录，鼠标离开后自动隐藏。
- 手机端：点击左边缘热区时显示目录。

页面切换时：

- 左侧目录应保持显示状态，除非用户主动隐藏。
- 目录显示/隐藏状态应写入 localStorage。

## 视觉风格

整体风格：

- 白色背景
- 浅灰边框
- 淡蓝强调色
- 少量阴影
- 轻量过渡动画

避免：

- 大面积渐变
- 闪烁动画
- 花哨背景
- 重型前端框架

## 字体建议

- 中文：Microsoft YaHei
- 英文：Segoe UI
- 代码：Consolas

正文默认 16px，代码默认 14px。

## 响应式要求

支持：

- 桌面端
- 平板端
- 手机端

手机端：

- 目录默认隐藏。
- 按钮触控区域不小于 40px。
- 内容区保持足够左右留白。

## 性能要求

- 首页尽量保持单文件或少文件结构。
- 不使用大型框架。
- 不依赖外部 CDN。
- SVG 图标内嵌或 CSS 绘制。
- 大型工具页面采用外链或 iframe 延迟加载。

## 可维护性

公共布局、颜色、间距、按钮、卡片等样式应集中维护。

新增工具或文章时，优先只修改导航数据和内容区数据，避免复制大量 HTML。

## 编码规范

- HTML 使用语义化标签。
- CSS 使用变量统一颜色和尺寸。
- JavaScript 保持短小、清晰、无复杂依赖。
- localStorage 键名应带站点前缀，避免冲突。
- 所有链接应明确 title 与说明。
